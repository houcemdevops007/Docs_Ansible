TP 06

Variables & Ansible Facts

Définir et utiliser des variables dans les playbooks, explorer les facts système collectés automatiquement et utiliser le module debug pour le diagnostic.

Intermédiaire 45 minutes

Objectifs

  • Définir des variables dans un playbook avec la directive vars
  • Utiliser la syntaxe Jinja2 {{ variable }} pour référencer les variables
  • Explorer les Ansible Facts collectés par le module setup
  • Filtrer les facts avec le paramètre filter
  • Utiliser le module debug pour afficher des informations

Prérequis

  • TPs 01 à 05 complétés

Étapes de réalisation

📝 Partie 1 : Variables utilisateur

1

Playbook avec variables définies dans vars

variable.yaml 
---
# Exemple d'utilisation des variables dans un playbook
# Les variables sont définies dans la section vars
- name: example avec les variables
  hosts: all
  become: true
  vars:
    - var1: "Hello DevOps"
    - var2: "Good Morning DevOps"
  tasks:
    # Utilisation des variables avec la syntaxe Jinja2 {{ }}
    - name: echo text
      shell: echo "var1 is {{ var1 }}, but var2 is {{ var2 }}" >> /home/ansadmin/example-variable.txt
ansadmin@server:~$ ansible-playbook variable.yaml TASK [echo text] ******************************************************** changed: [192.168.1.10] PLAY RECAP ************************************************************** 192.168.1.10 : ok=2 changed=1 ansadmin@server:~$ ssh ansadmin@192.168.1.10 "cat /home/ansadmin/example-variable.txt" var1 is Hello DevOps, but var2 is Good Morning DevOps

🔍 Partie 2 : Ansible Facts

2

Explorer les facts avec le module setup

Terminal 
# Afficher TOUS les facts d'un hôte (très verbeux)
ansible all -m setup

# Filtrer par famille d'OS
ansible all -m setup -a "filter=*family*"

# Filtrer les adresses IPv4
ansible all -m setup -a "filter=ansible_all_ipv4_addresses"
ansadmin@server:~$ ansible all -m setup -a "filter=*family*" 192.168.1.10 | SUCCESS => { "ansible_facts": { "ansible_os_family": "Debian" } }
3

Utiliser les facts dans un playbook

fact.yml 
---
# Utilisation des Ansible Facts dans un playbook
# Les facts sont collectés automatiquement au début du play (Gathering Facts)
- name: example avec les facts
  hosts: all
  become: true
  tasks:
    # Utiliser le fact ansible_os_family pour créer un fichier nommé dynamiquement
    - name: echo os family
      shell: echo "{{ ansible_os_family }}" >> /home/ansadmin/{{ ansible_os_family }}.txt

🐛 Partie 3 : Module debug

4

Afficher des informations avec debug

debug.yml 
---
# Utilisation du module debug pour afficher des informations
# Utile pour le diagnostic et la vérification des variables
- name: exemple debug
  hosts: all
  tasks:
    # Afficher un message avec des facts interpolés
    - name: afficher la version OS
      debug:
        msg: "Le système est {{ ansible_os_family }} version {{ ansible_distribution_version }}"
ansadmin@server:~$ ansible-playbook debug.yml TASK [afficher la version OS] ************************************************ ok: [192.168.1.10] => { "msg": "Le système est Debian version 22.04" }

Points clés à retenir

  • Les variables sont définies avec vars: et référencées avec {{ nom_variable }}.
  • Les Ansible Facts sont collectés automatiquement par le module setup au début de chaque play.
  • Facts courants : ansible_os_family, ansible_hostname, ansible_default_ipv4.address, ansible_distribution_version.
  • Le module debug est essentiel pour le diagnostic : il affiche des messages sans modifier les hôtes.
  • Le filtre setup (-a "filter=*pattern*") réduit la sortie aux facts pertinents.
