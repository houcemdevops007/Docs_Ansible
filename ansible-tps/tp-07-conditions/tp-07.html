TP 07

Conditions (when)

Utiliser les conditions when pour adapter les tâches selon le type d'OS, le résultat d'une tâche précédente ou toute autre expression logique.

Intermédiaire 40 minutes

Objectifs

  • Utiliser la directive when pour conditionner l'exécution des tâches
  • Combiner register et when pour réagir aux résultats de tâches
  • Utiliser ignore_errors pour continuer malgré les erreurs
  • Conditionner les actions selon la famille d'OS (ansible_os_family)

Prérequis

  • TPs 01 à 06 complétés, maîtrise des facts Ansible

Étapes de réalisation

🔄 Exemple 1 : Condition avec register + is failed

1

Tester l'existence d'un fichier et agir en conséquence

when.yaml 
---
# Condition avec register + is failed
# register capture le résultat de la tâche
# when vérifie si la tâche précédente a échoué
- name: exemple avec condition register
  hosts: all
  tasks:
    # Tester si le fichier existe (la commande échoue si absent)
    - name: tester si le fichier existe
      shell: ls /home/ansadmin/resultat.txt
      register: result
      ignore_errors: true

    # Créer le fichier uniquement s'il n'existe pas (échec de la tâche précédente)
    - name: créer le fichier si absent
      file:
        path: /home/ansadmin/resultat.txt
        state: touch
      when: result is failed
TASK [tester si le fichier existe] ****************************************** fatal: [192.168.1.10]: FAILED! => {"rc": 2, "stderr": "ls: cannot access..."} ...ignoring TASK [créer le fichier si absent] ******************************************* changed: [192.168.1.10] ← Le fichier a été créé car le test a échoué

🖥️ Exemple 2 : Condition selon l'OS

2

Installer le bon package selon la famille d'OS

when1.yaml 
---
# Condition when avec ansible_os_family
# Permet d'adapter les tâches selon le type d'OS
- name: exemple avec condition when os_family
  hosts: all
  become: true
  tasks:
    # Installer httpd sur les systèmes RedHat/CentOS
    - name: install httpd on centos
      yum:
        name: httpd
        state: latest
      when: ansible_os_family == "RedHat"

    # Installer apache2 sur les systèmes Debian/Ubuntu
    - name: install apache2 on ubuntu
      apt:
        name: apache2
        state: latest
      when: ansible_os_family == "Debian"
TASK [install httpd on centos] ********************************************** skipping: [192.168.1.10] ← Skipped car l'OS est Debian TASK [install apache2 on ubuntu] ******************************************** changed: [192.168.1.10] ← Exécuté car ansible_os_family == "Debian"

Points clés à retenir

  • when est une directive de tâche qui conditionne l'exécution selon une expression booléenne.
  • register capture le résultat d'une tâche dans une variable pour l'utiliser dans when.
  • ignore_errors: true empêche l'arrêt du playbook si la tâche échoue.
  • Tests Jinja2 : is failed, is succeeded, is changed, is skipped.
  • Pas de {{ }} dans les conditions when : on écrit directement when: ansible_os_family == "Debian".
