TP 10

Ansible Vault

Sécuriser vos données sensibles avec Ansible Vault. Apprenez à chiffrer des fichiers de variables, des mots de passe et des secrets pour les partager en toute sécurité dans vos dépôts de code.

Avancé 45 minutes

Objectifs

  • Comprendre le fonctionnement du chiffrement avec Ansible Vault
  • Créer et éditer des fichiers chiffrés
  • Chiffrer et déchiffrer des fichiers existants
  • Utiliser des variables chiffrées dans un playbook
  • Exécuter un playbook nécessitant un mot de passe Vault

Prérequis

  • TPs 01 à 09 complétés

Sécurité par Chiffrement

Fichier en clair (YAML) Ansible Vault (AES-256) ┌──────────────────────┐ ┌────────────────────────┐ │ db_password: "123" │──────────────▶│ $ANSIBLE_VAULT;1.1;AES │ │ api_key: "abc" │ CHIFREMENT │ 36393165353531393132... │ └──────────────────────┘ └────────────────────────┘ ▲ │ MOT DE PASSE VAULT ─┘

Étapes de réalisation

1

Commandes fondamentales Ansible Vault

Terminal 
# Créer un nouveau fichier chiffré
ansible-vault create vault.yml

# Chiffrer un fichier existant
ansible-vault encrypt vault.yml

# Déchiffrer un fichier
ansible-vault decrypt vault.yml

# Voir le contenu d'un fichier chiffré sans le déchiffrer
ansible-vault view vault.yml

# Éditer un fichier chiffré
ansible-vault edit vault.yml

# Changer le mot de passe Vault
ansible-vault rekey vault.yml
2

Préparer les données sensibles

Utilisez ansible-vault create vault.yml et insérez le contenu suivant :

vault.yml (En clair) 
# Contenu avant chiffrement
db_password: "SuperSecret123"
api_key: "abc123xyz456"
admin_password: "AdminPass2024"
3

Utiliser le Vault dans un Playbook

Le playbook utilise vars_files pour charger les variables chiffrées.

playbook-vault.yaml 
---
- name: exemple avec vault
  hosts: all
  become: true
  vars_files:
    - vault.yml
  tasks:
    # Afficher un message avec une variable chiffrée (à des fins de démo)
    - name: afficher le message avec variable chiffrée
      debug:
        msg: "Le mot de passe DB est {{ db_password }}"

    # Utiliser une variable vault dans un template
    - name: créer un fichier de config avec variable vault
      template:
        src: templates/config.j2
        dest: /etc/app/config.conf
4

Exécuter le playbook

Deux méthodes pour fournir le mot de passe :

Terminal 
# Méthode interactive (demande le mot de passe au clavier)
ansible-playbook playbook-vault.yaml --ask-vault-pass

# Méthode automatisée (utilise un fichier texte contenant le mot de passe)
# Note : le fichier ~/.vault_pass ne doit pas être versionné !
ansible-playbook playbook-vault.yaml --vault-password-file ~/.vault_pass
ansadmin@server:~$ ansible-playbook playbook-vault.yaml --ask-vault-pass Vault password: ******** ... TASK [afficher le message avec variable chiffrée] ************************ ok: [192.168.1.10] => { "msg": "Le mot de passe DB est SuperSecret123" }

Points clés à retenir

  • Ansible Vault chiffre les fichiers au repos via l'algorithme AES-256.
  • Fichiers vault : Ils ressemblent à des blocs de données binaires illisibles sur le disque.
  • --ask-vault-pass est indispensable pour demander le mot de passe à l'exécution.
  • Bonne pratique : Utiliser un fichier séparé (ex: vault.yml) pour les secrets et l'inclure via vars_files.
  • Sécurité : Ne versionnez jamais le mot de passe Vault en clair (.vault_pass).
