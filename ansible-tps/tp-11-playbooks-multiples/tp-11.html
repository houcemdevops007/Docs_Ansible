TP 11

Playbooks Multiples

Apprenez à structurer et modulariser vos projets Ansible en divisant vos playbooks. Découvrez comment importer d'autres playbooks et inclure des listes de tâches réutilisables.

Avancé 40 minutes

Objectifs

  • Comprendre l'importance de la modularité dans les grands projets
  • Utiliser import_playbook pour assembler plusieurs fichiers YAML
  • Utiliser include_tasks pour insérer des listes de tâches
  • Distinguer l'importation statique de l'inclusion dynamique

Prérequis

  • TPs 01 à 10 complétés, bonne compréhension des plays et des tâches

Architecture Modulaire

site.yaml (Playbook Master) ├── import_playbook: install-apache.yaml # Import statique └── hosts: tomcat tasks: └── include_tasks: tasks/install.yaml # Inclusion dynamique Dossier tasks/ └── install.yaml (Liste de tâches pure)

Étapes de réalisation

1

Créer le playbook secondaire (Apache)

install-apache.yaml 
---
# Playbook autonome d'installation Apache
- name: install apache web server
  hosts: webservers
  become: true
  tasks:
    - name: install apache2
      apt:
        name: apache2
        state: latest
    - name: start apache2
      service:
        name: apache2
        state: started
        enabled: yes
2

Créer un fichier de tâches externe (Tomcat)

Contrairement à un playbook, ce fichier ne contient qu'une liste de tâches (sans hosts/become).

tasks/install-task.yaml 
---
# Liste de tâches pure, destinée à être incluse
- name: install tomcat9
  apt:
    name: tomcat9
    state: latest
- name: start tomcat9
  service:
    name: tomcat9
    state: started
    enabled: yes
3

Créer le playbook principal (site.yaml)

site.yaml 
---
# Playbook Main qui orchestre tout le déploiement
# 1. Importation d'un playbook complet
- import_playbook: install-apache.yaml

# 2. Play pour Tomcat incluant un fichier de tâches
- name: install tomcat server
  hosts: tomcat
  become: true
  tasks:
    - include_tasks: tasks/install-task.yaml
4

Exécution du projet complet

Terminal 
# Lancer le playbook maître
ansible-playbook site.yaml
PLAY [install apache web server] ******************************************** ok: [192.168.1.10] ... PLAY [install tomcat server] ************************************************ TASK [include_tasks: tasks/install-task.yaml] ******************************* included: /home/ansadmin/tasks/install-task.yaml for 192.168.1.20 ...

Import vs Include

Caractéristique import_* (Statique) include_* (Dynamique)
Traitement Chargé au parsing du playbook Chargé pendant l'exécution
Visibilité Les tâches sont pré-listées au démarrage Le fichier est chargé quand la tâche arrive
Conditions Non affecté par when sur l'import Peut être sauté dynamiquement
Performance Meilleure (un seul chargement) Moins bonne (chargement à la demande)

Points clés à retenir

  • import_playbook assemble plusieurs playbooks complets.
  • include_tasks insère une liste de tâches au milieu d'un play.
  • Modularité : Permet de réutiliser le même code (ex: install-task) sur différents groupes d'hôtes.
  • Maintenance : Plus facile de corriger un petit fichier spécialisé qu'un playbook de 1000 lignes.
