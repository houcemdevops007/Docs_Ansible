TP 03

Commandes Ad-Hoc

Maîtriser les commandes ad-hoc Ansible pour exécuter des tâches rapides sans écrire de playbook : ping, copie de fichiers, gestion des utilisateurs et des packages.

Débutant 40 minutes

Objectifs

  • Comprendre la syntaxe des commandes ad-hoc Ansible
  • Utiliser les modules courants : ping, copy, user, file, apt, yum, service, shell
  • Comprendre l'option -b (become) pour l'élévation de privilèges
  • Exécuter des commandes rapides pour l'administration système

Prérequis

  • TP 01 et TP 02 complétés
  • Inventaire fonctionnel avec au moins un hôte accessible

Syntaxe générale

ansible <pattern> -m <module> -a "<arguments>" [options] pattern : all, webservers, 192.168.1.10, etc. -m : module à utiliser (ping, copy, shell, etc.) -a : arguments du module -b : become (élévation root via sudo) -i : fichier d'inventaire alternatif

Commandes Ad-Hoc par catégorie

🔍 Test de connectivité

1

Module ping

Vérifie la connectivité SSH et la présence de Python sur les hôtes cibles :

Terminal 
# Tester tous les hôtes de l'inventaire
ansible all -m ping

⚙️ Exécution de commandes

2

Commande système directe

Terminal 
# Redémarrer tous les hôtes (nécessite -b pour become root)
ansible all -a "/sbin/reboot" -b

# Exécuter une commande shell
ansible all -m shell -a "uname -a"

# Lister le contenu de /root (nécessite become)
ansible all -b -a "ls /root"

📂 Gestion de fichiers

3

Modules copy et file

Terminal 
# Copier un dossier vers les hôtes distants
ansible all -m copy -a "src=/home/dan dest=/tmp/home"

# Modifier les permissions d'un fichier
ansible all -m file -a "dest=/home/dan/file1.txt mode=777"

👤 Gestion des utilisateurs

4

Module user

Terminal 
# Créer un utilisateur avec mot de passe chiffré
ansible all -m user -a "name=testuser password=<encrypted_password>" -b

# Supprimer un utilisateur
ansible all -m user -a "name=testuser state=absent" -b
Pour générer un mot de passe chiffré : python3 -c "from passlib.hash import sha512_crypt; print(sha512_crypt.hash('MonMotDePasse'))"

📦 Gestion des packages

5

Modules apt et yum

Terminal 
# Installer un package avec yum (CentOS/RedHat)
ansible all -m yum -a "name=httpd state=latest" -b

# Installer un package avec apt (Ubuntu/Debian)
ansible all -m apt -a "name=nginx state=latest" -b

🔧 Gestion des services

6

Module service

Terminal 
# Démarrer un service
ansible all -m service -a "name=httpd state=started" -b

# Arrêter un service
ansible all -m service -a "name=nginx state=stopped" -b

Résultat attendu

ansadmin@server:~$ ansible all -m shell -a "uname -a" 192.168.1.10 | CHANGED | rc=0 >> Linux client1 5.15.0-91-generic #101-Ubuntu SMP x86_64 GNU/Linux 192.168.1.11 | CHANGED | rc=0 >> Linux client2 5.15.0-91-generic #101-Ubuntu SMP x86_64 GNU/Linux

Points clés à retenir

  • Les commandes ad-hoc sont idéales pour les tâches ponctuelles et le diagnostic rapide.
  • L'option -b (become) est nécessaire pour les tâches nécessitant des privilèges root.
  • Le module par défaut est command quand on n'utilise pas -m.
  • Préférez les playbooks pour les tâches répétitives ou complexes (idempotence, documentation).
  • Le module shell permet d'utiliser des pipes et redirections, contrairement à command.
