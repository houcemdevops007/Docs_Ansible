TP 12

Roles Ansible & Galaxy

Le niveau ultime d'organisation Ansible. Apprenez à créer des roles réutilisables, à structurer vos projets selon les standards professionnels et à consommer des roles communautaires via Ansible Galaxy.

Expert 60 minutes

Objectifs

  • Comprendre la structure standard d'un Role Ansible
  • Créer un role personnalisé "apache" de A à Z
  • Utiliser les dossiers tasks, handlers, vars, defaults et templates
  • Rechercher et installer des roles via ansible-galaxy
  • Appeler un role depuis un playbook principal

Prérequis

  • Tous les TPs précédents (01 à 11) validés

Architecture d'un Role

roles/ └── apache/ ├── defaults/ # Variables par défaut (basse priorité) ├── files/ # Fichiers statiques à copier ├── handlers/ # Handlers spécifiques au role ├── meta/ # Métadonnées du role (auteur, dépendances) ├── tasks/ # Liste principale des tâches ├── templates/# Templates Jinja2 ├── tests/ # Inventaire et playbook de test └── vars/ # Variables du role (haute priorité)

Étapes de réalisation

🛠️ Partie 1 : Créer un role personnalisé

1

Initialiser la structure du role

Terminal 
# Créer le dossier roles s'il n'existe pas
mkdir roles
cd roles

# Utiliser ansible-galaxy pour générer le squelette
ansible-galaxy init apache
2

Définir les tâches (tasks/main.yml)

roles/apache/tasks/main.yml 
---
- name: install apache package
  apt:
    name: "{{ apache_package }}"
    state: latest

- name: deploy custom index.html
  template:
    src: index.html.j2
    dest: /var/www/html/index.html
  notify: restart_apache

- name: ensure apache is running
  service:
    name: "{{ apache_service }}"
    state: started
    enabled: yes
3

Définir les variables par défaut (defaults/main.yml)

roles/apache/defaults/main.yml 
---
# Valeurs par défaut écrasables facilement
apache_package: apache2
apache_service: apache2
http_port: 80
4

Appeler le role dans un playbook

site.yml 
---
- name: deploy web servers using roles
  hosts: webservers
  become: true
  roles:
    - apache

🌌 Partie 2 : Ansible Galaxy

5

Utiliser des roles communautaires

Terminal 
# Rechercher un role pour MySQL
ansible-galaxy search mysql

# Installer le role Apache de Jeff Geerling
ansible-galaxy install geerlingguy.apache

Points clés à retenir

  • Un Role est une unité autonome qui regroupe tout ce qui est nécessaire à une fonction (ex: configurer une DB).
  • Haute réutilisabilité : On peut copier le dossier d'un role d'un projet à un autre.
  • Priorité des variables : Les variables dans vars/ gagnent sur defaults/.
  • Ansible Galaxy : Le "Play Store" d'Ansible pour télécharger des solutions éprouvées.
