TP 01

Installation & Configuration SSH

Installer Ansible sur Ubuntu, créer l'utilisateur d'administration et configurer l'authentification SSH par clé entre le serveur de contrôle et les clients.

Débutant 45 minutes

Objectifs

  • Installer Ansible sur une machine Ubuntu (serveur de contrôle)
  • Vérifier la version et la configuration par défaut d'Ansible
  • Créer un utilisateur dédié ansadmin avec privilèges sudo sans mot de passe
  • Configurer l'authentification SSH par clé publique/privée
  • Distribuer la clé SSH vers les machines clientes

Prérequis

  • 2 machines virtuelles Ubuntu 20.04 ou 22.04 LTS (1 serveur + 1 client minimum)
  • Accès root ou sudo sur les deux machines
  • Connectivité réseau entre les machines (même réseau ou VPN)
  • Python 3 installé (présent par défaut sur Ubuntu)

Schéma d'architecture

┌──────────────────────────┐ SSH (clé) ┌──────────────────────────┐ │ SERVEUR DE CONTRÔLE │ ──────────────────────────▶ │ CLIENT (Node 1) │ │ (Ansible installé) │ │ │ │ │ SSH (clé) ├──────────────────────────┤ │ Ubuntu 22.04 LTS │ ──────────────────────────▶ │ CLIENT (Node 2) │ │ IP: 192.168.1.100 │ │ │ │ User: ansadmin │ │ Ubuntu 22.04 LTS │ │ │ │ IP: 192.168.1.10/11 │ │ ~/.ssh/id_rsa (privée) │ │ User: ansadmin │ │ ~/.ssh/id_rsa.pub │ │ ~/.ssh/authorized_keys │ └──────────────────────────┘ └──────────────────────────┘

Étapes de réalisation

1

Mettre à jour le système et installer Ansible

Sur le serveur de contrôle, mettez à jour les dépôts et installez Ansible :

Terminal — Serveur 
# Mettre à jour les dépôts
sudo apt update

# Installer Ansible
sudo apt install ansible -y
2

Vérifier l'installation

Confirmez que Ansible est correctement installé en affichant sa version :

Terminal — Serveur 
ansible --version
ansadmin@server:~$ ansible --version ansible [core 2.14.3] config file = /etc/ansible/ansible.cfg configured module search path = ['/home/ansadmin/.ansible/plugins/modules'] ansible python module location = /usr/lib/python3/dist-packages/ansible ansible collection location = /home/ansadmin/.ansible/collections executable location = /usr/bin/ansible python version = 3.10.12
3

Créer l'utilisateur ansadmin

Créez un utilisateur dédié à Ansible sur toutes les machines (serveur et clients) :

Terminal — Toutes les machines 
# Créer l'utilisateur ansadmin
sudo adduser ansadmin
Suivez les invites pour définir le mot de passe et les informations du profil. Retenez bien le mot de passe, il sera utilisé temporairement pour la copie de la clé SSH.
4

Configurer sudo sans mot de passe

Ajoutez la règle sudoers sur toutes les machines pour permettre à ansadmin d'exécuter des commandes root sans mot de passe :

Terminal — Toutes les machines 
# Éditer le fichier sudoers (méthode sûre avec visudo)
sudo visudo

# Ajouter cette ligne à la fin du fichier :
ansadmin ALL=(ALL) NOPASSWD:ALL
Important : Utilisez toujours visudo pour éditer le fichier sudoers. Une erreur de syntaxe peut rendre la commande sudo inutilisable.
5

Activer l'authentification par mot de passe SSH (côté client)

Sur chaque machine cliente, activez temporairement l'authentification par mot de passe pour permettre la copie de la clé SSH :

/etc/ssh/sshd_config — Client 
# Éditer la configuration SSH
sudo nano /etc/ssh/sshd_config

# Trouver et modifier cette ligne :
PasswordAuthentication yes

# Recharger le service SSH
sudo systemctl reload sshd
6

Générer la paire de clés SSH

Sur le serveur de contrôle, connectez-vous en tant qu'ansadmin et générez la paire de clés :

Terminal — Serveur (ansadmin) 
# Se connecter en tant qu'ansadmin
su - ansadmin

# Générer la paire de clés RSA (appuyer sur Entrée pour accepter les valeurs par défaut)
ssh-keygen -t rsa
ansadmin@server:~$ ssh-keygen -t rsa Generating public/private rsa key pair. Enter file in which to save the key (/home/ansadmin/.ssh/id_rsa): [Entrée] Enter passphrase (empty for no passphrase): [Entrée] Enter same passphrase again: [Entrée] Your identification has been saved in /home/ansadmin/.ssh/id_rsa Your public key has been saved in /home/ansadmin/.ssh/id_rsa.pub
7

Copier la clé SSH vers les clients

Distribuez la clé publique vers chaque machine cliente. Remplacez <IP_CLIENT> par l'adresse IP réelle :

Terminal — Serveur (ansadmin) 
# Copier la clé vers le client (entrer le mot de passe ansadmin du client)
ssh-copy-id ansadmin@192.168.1.10

# Vérifier la connexion sans mot de passe
ssh ansadmin@192.168.1.10
ansadmin@server:~$ ssh-copy-id ansadmin@192.168.1.10 /usr/bin/ssh-copy-id: INFO: Source of key(s) to be installed: "/home/ansadmin/.ssh/id_rsa.pub" Number of key(s) added: 1 Now try logging into the machine, with: "ssh 'ansadmin@192.168.1.10'"

Résultat attendu

Après avoir complété toutes les étapes, vous devriez pouvoir :

ansadmin@server:~$ ssh ansadmin@192.168.1.10 Welcome to Ubuntu 22.04.3 LTS (GNU/Linux 5.15.0-91-generic x86_64) Last login: Thu Mar 12 08:00:00 2026 from 192.168.1.100 ansadmin@client:~$ ← Connexion sans mot de passe réussie !

Points clés à retenir

  • Ansible est agentless : il utilise SSH pour communiquer avec les machines cibles, aucun agent à installer sur les clients.
  • L'authentification par clé SSH est essentielle pour l'automatisation : pas d'interaction manuelle pour les mots de passe.
  • L'utilisateur ansadmin est une bonne pratique : séparer l'administration Ansible des comptes personnels.
  • sudo NOPASSWD permet à Ansible d'exécuter des tâches nécessitant des privilèges root sans intervention.
  • Toujours utiliser visudo pour éditer /etc/sudoers afin d'éviter les erreurs de syntaxe.
