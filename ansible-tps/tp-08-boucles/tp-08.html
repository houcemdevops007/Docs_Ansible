TP 08

Boucles (Loops)

Utiliser les boucles with_items, with_sequence et les boucles modernes loop pour automatiser des tâches répétitives comme la création de fichiers ou l'installation de listes de packages.

Intermédiaire 35 minutes

Objectifs

  • Utiliser with_items pour itérer sur une liste simple
  • Utiliser with_sequence pour générer des itérations numériques
  • Itérer sur des listes de variables définies en YAML
  • Comprendre l'avantage des boucles par rapport à la répétition de tâches

Prérequis

  • TPs 01 à 07 complétés

Étapes de réalisation

📂 Exemple 1 : with_items

1

Création de fichiers en série

items.yaml 
---
- name: exemple avec items
  hosts: all
  tasks:
    - name: creation de liste des fichiers
      file:
        path: "{{ item }}"
        state: touch
        mode: "0775"
      with_items:
        - init1.txt
        - init2.txt
        - init3.txt
        - init4.txt

📦 Exemple 2 : Itérer sur une liste de variables

2

Installation de plusieurs packages

list.yml 
---
- name: exemple ansible list
  hosts: all
  become: true
  vars:
    OtherPackages:
      - vim
      - curl
      - git
  tasks:
    - name: install packages
      apt:
        name: "{{ OtherPackages }}"
        state: present
Note : Le module apt supporte nativement les listes. Passer une liste au paramètre name est plus performant qu'une boucle with_items car Ansible effectue une seule transaction APT.

🔢 Exemple 3 : with_sequence

3

Générer des ressources numérotées

seq.yaml 
---
- name: exemple avec sequence
  hosts: all
  tasks:
    - name: creation de liste des fichiers numerotes
      file:
        path: "DevOps{{ item }}.txt"
        state: touch
        mode: "0775"
      with_sequence: start=20 end=27

Résultat attendu

ansadmin@server:~$ ansible-playbook seq.yaml TASK [creation de liste des fichiers numerotes] ******************************* changed: [192.168.1.10] => (item=20) changed: [192.168.1.10] => (item=21) ... changed: [192.168.1.10] => (item=27)

Points clés à retenir

  • with_items itère sur une liste. La valeur de l'itération actuelle est disponible dans {{ item }}.
  • with_sequence génère des nombres sur une plage donnée (start, end, stride, format).
  • Boucle native : Certains modules (apt, yum, copy) acceptent directement des listes sans avoir besoin de with_items.
  • loop : C'est la syntaxe moderne recommandée par Ansible pour remplacer les directives with_X.
