TP 02

Inventaire Ansible

Configurer l'inventaire statique en formats INI et YAML avec des groupes d'hôtes pour organiser votre infrastructure.

Débutant 30 minutes

Objectifs

  • Comprendre le rôle de l'inventaire dans Ansible
  • Créer un inventaire statique au format INI
  • Créer un inventaire statique au format YAML
  • Organiser les hôtes par groupes logiques (webservers, tomcat, dbservers)
  • Tester la connectivité avec le module ping

Prérequis

  • TP 01 complété : Ansible installé, SSH configuré avec ansadmin
  • Au moins 2 machines cibles accessibles en SSH

Schéma d'architecture

┌─────────────────────┐ │ ANSIBLE SERVER │ │ ansible.cfg │──────┐ │ inventory/hosts │ │ │ inventory/hosts.yml │ └─────────────────────┘ │ │ ┌───────────────────┼───────────────────┐ ▼ ▼ ▼ ┌─────────────┐ ┌─────────────┐ ┌─────────────┐ │ webservers │ │ tomcat │ │ dbservers │ │ │ │ │ │ │ │ 192.168.1.10│ │ 192.168.1.20│ │ 192.168.1.30│ │ 192.168.1.11│ │ │ │ │ └─────────────┘ └─────────────┘ └─────────────┘

Étapes de réalisation

1

Créer la structure du projet

Terminal 
# Créer le dossier du TP et le sous-dossier inventory
mkdir -p tp-02-inventaire/inventory
cd tp-02-inventaire
2

Créer l'inventaire au format INI

Le format INI est le format classique et le plus simple pour les inventaires Ansible :

inventory/hosts (INI) 
# Groupe des serveurs web (Apache/Nginx)
[webservers]
192.168.1.10 ansible_user=ansadmin
192.168.1.11 ansible_user=ansadmin

# Groupe du serveur d'applications Tomcat
[tomcat]
192.168.1.20 ansible_user=ansadmin

# Groupe des serveurs de base de données
[dbservers]
192.168.1.30 ansible_user=ansadmin

# Variables communes à tous les hôtes
[all:vars]
ansible_ssh_private_key_file=~/.ssh/id_rsa
3

Créer l'inventaire au format YAML

Le format YAML offre plus de lisibilité et de structuration :

inventory/hosts.yml (YAML) 
# Inventaire Ansible au format YAML
# Structure hiérarchique avec groupes et sous-groupes
all:
  children:
    # Groupe des serveurs web
    webservers:
      hosts:
        web1:
          ansible_host: 192.168.1.10
          ansible_user: ansadmin
        web2:
          ansible_host: 192.168.1.11
          ansible_user: ansadmin

    # Groupe du serveur Tomcat
    tomcat:
      hosts:
        app1:
          ansible_host: 192.168.1.20
          ansible_user: ansadmin

    # Groupe des serveurs de base de données
    dbservers:
      hosts:
        db1:
          ansible_host: 192.168.1.30
          ansible_user: ansadmin
4

Configurer ansible.cfg

Créez un fichier de configuration local pour pointer vers votre inventaire par défaut :

ansible.cfg 
[defaults]
# Chemin vers le fichier d'inventaire par défaut
inventory = inventory/hosts
# Désactiver la vérification des clés SSH (lab uniquement)
host_key_checking = False
# Utilisateur distant par défaut
remote_user = ansadmin
5

Tester la connectivité

Utilisez le module ping pour vérifier la connexion vers tous les hôtes :

Terminal 
# Tester tous les hôtes
ansible all -m ping

# Tester un groupe spécifique
ansible webservers -m ping

# Tester avec un inventaire ad-hoc (liste d'IPs)
ansible -i "192.168.1.10,192.168.1.11," -m ping all
ansadmin@server:~$ ansible all -m ping 192.168.1.10 | SUCCESS => { "ansible_facts": { "discovered_interpreter_python": "/usr/bin/python3" }, "changed": false, "ping": "pong" } 192.168.1.11 | SUCCESS => { "changed": false, "ping": "pong" }
La réponse "ping": "pong" confirme que la connexion SSH et Python fonctionnent correctement sur le client.

Résultat attendu

Tous les hôtes de votre inventaire répondent SUCCESS avec "ping": "pong". Vous pouvez cibler des groupes spécifiques pour des tâches ciblées.

Points clés à retenir

  • Deux formats d'inventaire : INI (simple) et YAML (structuré). Choisissez selon la complexité de votre infrastructure.
  • Les groupes permettent de cibler des sous-ensembles de machines (webservers, dbservers).
  • Le groupe all englobe automatiquement tous les hôtes définis.
  • [all:vars] définit des variables communes à toutes les machines.
  • ansible.cfg dans le répertoire courant a la priorité sur /etc/ansible/ansible.cfg.
