TP 05

Handlers Ansible

Comprendre et utiliser les handlers pour déclencher des actions (comme le redémarrage d'un service) uniquement lorsqu'un changement a été effectué.

Intermédiaire 35 minutes

Objectifs

  • Comprendre le concept de handler et la directive notify
  • Créer un playbook avec handler pour redémarrer Nginx après modification
  • Observer l'idempotence : le handler ne s'exécute que si un changement a eu lieu
  • Utiliser le module template pour copier des fichiers

Prérequis

  • TPs 01 à 04 complétés, Nginx installé sur les hôtes cibles

Fonctionnement des Handlers

tasks: handlers: ┌─────────────────────┐ ┌─────────────────────┐ │ install nginx │ │ restart_nginx │ │ (changed/ok) │ │ service: │ └─────────────────────┘ │ name: nginx │ │ state: restarted │ ┌─────────────────────┐ notify └─────────────────────┘ │ copy index.html │──────────────────▶ (exécuté à la fin │ (changed → notify) │ du play si notifié) └─────────────────────┘ ⚠️ Un handler n'est exécuté que si : 1. La tâche qui le notifie a le statut "changed" 2. Il est exécuté UNE SEULE FOIS à la fin du play, même s'il est notifié plusieurs fois

Étapes de réalisation

1

Créer le fichier index.html

Ce fichier sera copié vers les serveurs web cibles :

index.html 
<html>
  <body>
    <h1>Serveur configuré par Ansible - TP Handlers</h1>
  </body>
</html>
2

Créer le playbook avec handler

nginx-handler.yaml 
---
# Playbook avec handler : le restart ne se fait que si le fichier a changé
- name: Configure webserver with nginx
  hosts: all
  become: True
  tasks:
    # Installer nginx (mise à jour du cache apt)
    - name: install nginx
      apt:
        name: nginx
        update_cache: yes

    # Copier le fichier index.html vers le serveur
    # Si le fichier change → notify déclenche le handler
    - name: copy index.html
      template:
        src: index.html
        dest: /var/www/html/index.html
      notify: restart_nginx

  # Les handlers sont déclenchés uniquement si notifiés
  handlers:
    - name: restart_nginx
      service:
        name: nginx
        state: restarted
3

Première exécution (changement détecté)

Terminal 
ansible-playbook nginx-handler.yaml
TASK [copy index.html] **************************************************** changed: [192.168.1.10] RUNNING HANDLER [restart_nginx] ******************************************* changed: [192.168.1.10] ← Le handler s'exécute car le fichier a été modifié
4

Deuxième exécution (aucun changement)

Relancez le même playbook pour voir l'idempotence en action :

Terminal 
ansible-playbook nginx-handler.yaml
TASK [copy index.html] **************************************************** ok: [192.168.1.10] ← Fichier inchangé, pas de notify PLAY RECAP **************************************************************** 192.168.1.10 : ok=3 changed=0 ← Le handler ne s'est PAS exécuté
Idempotence : Le handler restart_nginx ne s'est pas exécuté la 2e fois car la tâche copy index.html n'a pas modifié le fichier (état déjà conforme).

Points clés à retenir

  • Les handlers sont des tâches spéciales déclenchées par notify.
  • Un handler n'est exécuté que si la tâche qui le notifie a le statut changed.
  • Les handlers s'exécutent à la fin du play, après toutes les tâches — pas immédiatement après le notify.
  • Un handler notifié plusieurs fois ne s'exécute qu'une seule fois.
  • Cas d'usage typique : redémarrer un service après modification de sa configuration.
