TP 04

Premiers Playbooks

Écrire et exécuter vos premiers playbooks YAML pour installer Nginx et lancer des commandes shell de manière automatisée.

Débutant 45 minutes

Objectifs

  • Comprendre la structure d'un playbook YAML
  • Créer un playbook pour installer et démarrer Nginx
  • Créer un playbook utilisant le module shell
  • Comprendre la directive become: true pour l'élévation de privilèges
  • Exécuter un playbook avec ansible-playbook

Prérequis

  • TPs 01 à 03 complétés
  • Inventaire fonctionnel

Structure d'un Playbook

playbook.yaml ├── name: "Nom du play" # Description du play ├── hosts: all # Cible (groupe ou hôtes) ├── become: true # Élévation de privilèges (sudo) └── tasks: # Liste ordonnée de tâches ├── name: "Tâche 1" # Description │ module: # Module Ansible à utiliser │ paramètre: valeur # Arguments du module └── name: "Tâche 2" module: paramètre: valeur

Étapes de réalisation

1

Playbook d'installation Nginx

Créez le fichier nginx.yaml pour installer et démarrer Nginx sur toutes les machines :

nginx.yaml 
---
# Playbook d'installation et démarrage de Nginx
# Nécessite les privilèges root (become: true)
- name: Install nginx
  hosts: all
  become: true
  tasks:
    # Installer la dernière version de nginx via apt
    - name: Install nginx is at the latest version
      apt:
        name: nginx
        state: latest

    # Démarrer le service nginx
    - name: start nginx
      service:
        name: nginx
        state: started
2

Exécuter le playbook Nginx

Terminal 
# Exécuter le playbook
ansible-playbook nginx.yaml
ansadmin@server:~$ ansible-playbook nginx.yaml PLAY [Install nginx] ******************************************************** TASK [Gathering Facts] ****************************************************** ok: [192.168.1.10] ok: [192.168.1.11] TASK [Install nginx is at the latest version] ******************************* changed: [192.168.1.10] changed: [192.168.1.11] TASK [start nginx] ********************************************************** changed: [192.168.1.10] changed: [192.168.1.11] PLAY RECAP ****************************************************************** 192.168.1.10 : ok=3 changed=2 unreachable=0 failed=0 skipped=0 192.168.1.11 : ok=3 changed=2 unreachable=0 failed=0 skipped=0
3

Playbook avec commandes shell (sans become)

Créez shell.yml pour exécuter des commandes shell en tant qu'utilisateur normal :

shell.yml 
---
# Playbook exemple sans élévation de privilèges
# Les commandes s'exécutent en tant qu'ansadmin
- name: exemple sans become
  hosts: all
  tasks:
    # Récupérer la version du système d'exploitation
    - name: version OS
      shell: uname -a >> /home/ansadmin/resultat.txt

    # Identifier l'utilisateur courant
    - name: c'est qui??
      shell: whoami >> /home/ansadmin/resultat.txt
4

Exécuter le playbook shell

Terminal 
# Exécuter sans become (utilisateur ansadmin)
ansible-playbook shell.yml

# Exécuter avec become via la ligne de commande
ansible-playbook shell.yml -b
Différence clé : become: true dans le playbook = toujours en root. L'option -b en ligne de commande = pontuel, sans modifier le fichier.

Points clés à retenir

  • Un playbook est un fichier YAML contenant un ou plusieurs "plays" (ensembles de tâches).
  • become: true dans le playbook applique sudo à toutes les tâches du play.
  • -b en CLI permet d'activer become ponctuellement sans modifier le playbook.
  • L'idempotence : relancer le même playbook ne change rien si l'état est déjà atteint (changed → ok).
  • Le PLAY RECAP résume le nombre de tâches ok, changed, failed pour chaque hôte.
