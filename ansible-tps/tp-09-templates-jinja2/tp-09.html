TP 09

Templates Jinja2

Créer des fichiers de configuration dynamiques et personnalisés à l'aide du moteur de templating Jinja2, en intégrant des variables, des conditions et des boucles directement dans vos fichiers.

Intermédiaire 50 minutes

Objectifs

  • Comprendre le fonctionnement du moteur Jinja2 avec Ansible
  • Créer des templates .j2 utilisant des variables et des facts
  • Utiliser les structures de contrôle Jinja2 (if/else, for loops)
  • Déployer des fichiers de configuration dynamiques via le module template

Prérequis

  • TPs 01 à 08 complétés, maîtrise des variables et facts

Flux de données Template

Étapes de réalisation

1

Créer le template Jinja2

Le template utilise des doubles accolades {{ }} pour les variables et des accolades-pourcents {% %} pour les expressions de contrôle.

templates/index.html.j2 
<html>
  <body>
    <h2>This computer hostname is {{ ansible_hostname }}</h2>
    <h2>The OS of this computer is {{ ansible_os_family }}</h2>
    <h2>IP Address: {{ ansible_default_ipv4.address }}</h2>
    <h2>This file version is {{ file_version }}</h2>

    {% if ansible_os_family == "Debian" %}
      <p>Système Ubuntu/Debian détecté</p>
    {% else %}
      <p>Système RedHat/CentOS détecté</p>
    {% endif %}

    <h3>Interfaces Réseau :</h3>
    <ul>
    {% for interface in ansible_interfaces %}
      <li>{{ interface }}</li>
    {% endfor %}
    </ul>
  </body>
</html>
2

Créer le playbook de déploiement

template.yaml 
---
# Utilisation du module template pour générer un fichier dynamique
- name: exemple avec template
  hosts: all
  become: true
  vars:
    file_version: "2.0"
  tasks:
    # Le module template traite le fichier .j2 avant de le copier
    - name: ajout index.html dynamique
      template:
        src: templates/index.html.j2
        dest: /var/www/html/index.html
        mode: "0777"
3

Exécuter et vérifier

Terminal 
# Lancer le playbook
ansible-playbook template.yaml

# Vérifier le contenu généré sur le client
curl http://localhost
<html> <body> <h2>This computer hostname is client-ubuntu-01</h2> <h2>The OS of this computer is Debian</h2> <h2>IP Address: 192.168.1.10</h2> ...

Points clés à retenir

  • Jinja2 permet de rendre les fichiers statiques dynamiques.
  • Variables : {{ variable_name }} injecte la valeur.
  • Conditions : {% if %} ... {% else %} ... {% endif %}.
  • Boucles : {% for item in list %} ... {% endfor %}.
  • Module template : Contrairement à copy, il interprète le code Jinja2 avant le transfert.
